Замыкание (Closure) — это комбинация функции и лексического окружения (lexical environment), в котором эта функция была объявлена. Проще говоря, замыкание дает функции доступ к переменным из ее внешней области видимости (scope) даже после того, как внешняя функция завершила свое выполнение.

Как это работает:  
  Когда функция создается, она "запоминает" ссылки на переменные, которые были доступны в момент ее создания. Это лексическое окружение сохраняется вместе с функцией.

* JavaScript
```js
function outerFunction(outerVariable) {  
  return function innerFunction(innerVariable) {  
    // innerFunction имеет доступ к outerVariable (из outerFunction)  
    // и к innerVariable (своей собственной)  
    // и к глобальным переменным  
    console.log("outerVariable:", outerVariable);  
    console.log("innerVariable:", innerVariable);  
  };  
}

const newFunction \= outerFunction("outside"); // outerFunction выполнилась и вернула innerFunction  
newFunction("inside"); // Вызываем innerFunction. Она все еще "помнит"
outerVariable ("outside")  
// Вывод:  
// outerVariable: outside  
// innerVariable: inside
```

Здесь newFunction является замыканием. Она "замкнула" в себе переменную outerVariable.

**Сценарии использования замыканий:**

  1. Приватные переменные и методы (Модульный паттерн / IIFE):  
     Замыкания позволяют создавать переменные, доступные только внутри функции и ее вложенных функций, эмулируя приватность.

	```js
	const counter = (function() {
		let privateCount = 0; // Приватная переменная
		function changeBy(val) {  
		    privateCount += val;  
		}
		  
  return {  
    increment: function() { changeBy(1); },  
    decrement: function() { changeBy(-1); },  
    value: function() { return privateCount; }  
  };  
})();

console.log(counter.value()); // 0  
counter.increment();  
counter.increment();  
console.log(counter.value()); // 2  
// console.log(counter.privateCount); // undefined, нет прямого доступа
	```
   
   3. Каррирование (Currying) и частичное применение функций:  
      Возможность создавать функции, которые принимают аргументы по частям.

   4. JavaScript

function multiply(a) {  
  return function(b) { // Эта внутренняя функция \- замыкание  
    return a \* b;  
  };  
}  
const multiplyByTwo \= multiply(2); // multiplyByTwo "помнит" a \= 2  
console.log(multiplyByTwo(5)); // 10  
console.log(multiply(3)(4));   // 12

   7. Функции обратного вызова (Callbacks) с сохранением состояния:  
      Например, в обработчиках событий или setTimeout, где колбэк должен помнить какое-то состояние из момента его создания.

   8. JavaScript

for (var i \= 1; i \<= 3; i++) { // Проблема с var и замыканиями в цикле  
  setTimeout(function() {  
    // console.log(i); // Выведет 4, 4, 4 из\-за того, как var работает с замыканиями в цикле  
  }, i \* 100);  
}

// Решение с let (которое создает блочную область видимости для каждой итерации)  
for (let j \= 1; j \<= 3; j++) {  
  setTimeout(function() {  
    console.log(j); // Выведет 1, 2, 3  
  }, j \* 100);  
}

// Решение с замыканием (IIFE) для var  
for (var k \= 1; k \<= 3; k++) {  
  (function(savedK) {  
    setTimeout(function() {  
      console.log(savedK); // Выведет 1, 2, 3  
    }, savedK \* 100);  
  })(k);  
}

   10. Сохранение состояния между вызовами функции:  
       Функция может "помнить" информацию из предыдущих вызовов.

   11. JavaScript

function createUniqueIdGenerator() {  
  let id \= 0;  
  return function() {  
    id \+= 1;  
    return id;  
  };  
}  
const nextId \= createUniqueIdGenerator();  
console.log(nextId()); // 1  
console.log(nextId()); // 2
