var, let и const — это ключевые слова для объявления переменных, но они имеют важные различия в области видимости, всплытии (hoisting) и возможности переприсваивания.

| Характеристика | Var | Let | Const |
| :---- | :---- | :---- | :---- |
| Область видимости | Функциональная или глобальная | Блочная ({}) | Блочная ({}) |
| Всплытие (Hoisting) | Всплывает и инициализируется undefined | Всплывает, но не инициализируется (TDZ\*) | Всплывает, но не инициализируется (TDZ\*) |
| Переобъявление | Можно в той же области видимости | Нельзя в той же области видимости | Нельзя в той же области видимости |
| Переприсваивание | Можно | Можно | Нельзя (для примитивов; для объектов — см. ниже) |
| Создание свойства на глобальном объекте | Да, если объявлена в глобальной области | Нет | Нет |

 \* **TDZ (Temporal Dead Zone / Временная мертвая зона):** Переменные, объявленные с let и const, не могут быть доступны до их объявления в коде. Попытка доступа вызовет ReferenceError.

**const и объекты/массивы**  
Когда const используется с объектами или массивами, сама ссылка на объект/массив является константой и не может быть изменена (т.е. нельзя присвоить другую ссылку). Однако содержимое объекта или массива (его свойства или элементы) можно изменять.

const person \= { name: "Alice" };  
person.name \= "Bob"; // Это допустимо  
// person \= { name: "Charlie" }; // Ошибка: Assignment to constant variable.

const arr \= \[1, 2\];  
arr.push(3); // Это допустимо  
// arr \= \[4, 5\]; // Ошибка: Assignment to constant variable.

**Рекомендация:** Предпочитайте const по умолчанию. Используйте let только тогда, когда знаете, что переменной потребуется переприсваивание. Избегайте var в современном JavaScript из\-за его менее предсказуемого поведения с областью видимости и всплытием.
